## 🛡️ Cybersecurity

- **CVE-2026-10561 — unauthenticated remote code execution in IBM Langflow OSS scores a perfect CVSS 10.0 (the maximum; trivially exploitable, full host takeover).** Langflow is a popular open-source visual builder for LLM and agent workflows. The flaw pairs an authentication bypass with improper isolation of Python execution, so an unauthenticated network attacker can run arbitrary code on the host and fully compromise it — affected versions 1.0.0 through 1.9.3. A companion bug published the same day, **CVE-2026-7664 (CVSS 9.8)**, lets unauthenticated attackers reach protected MCP (Model Context Protocol — the open standard for wiring tools and data into AI agents) project resources and run MCP operations via the Streamable MCP transport endpoint, in versions up to 1.8.4. This is at least the third Langflow RCE the desk has logged in a month (after the May archive-extraction symlink bug, CVE-2026-7524, a *distinct* flaw) — a pattern worth noting for anyone self-hosting agent tooling. Why it matters: Langflow instances are exactly the kind of internet-exposed AI infrastructure being stood up fast and secured slowly; a CVSS 10.0 pre-auth RCE means patch now (≥1.9.4) or take it off the network. ([NVD CVE-2026-10561](https://nvd.nist.gov/vuln/detail/CVE-2026-10561); [NVD CVE-2026-7664](https://nvd.nist.gov/vuln/detail/CVE-2026-7664); [IBM advisory](https://www.ibm.com/support/pages/node/7277242))
- **MISP ships a coordinated batch of critical access-control fixes — the threat-intel platform patching its own authorization.** MISP (Malware Information Sharing Platform — the de-facto open-source standard for sharing indicators of compromise between security teams) disclosed a cluster of broken-access-control flaws. The lead, **CVE-2026-56422 (CVSS 9.4)**, stems from controllers accepting client-controlled fields — primary keys and ownership/scope identifiers like `event_id`, `org_id`, `sharing_group_id` — without revalidating them, letting a user reassign or reach records across organisational boundaries (an *insecure direct object reference*). Siblings include **CVE-2026-56423 (9.4)**, broken authorization in bulk-deletion of event reports and sharing groups, and **CVE-2026-56425 (9.3)**, weaknesses in the Azure AD (Microsoft's cloud identity service) OAuth 2.0 login flow. Why it matters: MISP underpins inter-org threat sharing in many SOCs (security operations centres), so cross-tenant access flaws here risk leaking the very indicator data that's supposed to be tightly scoped — upgrade MISP core promptly. ([NVD CVE-2026-56422](https://nvd.nist.gov/vuln/detail/CVE-2026-56422); [NVD CVE-2026-56425](https://nvd.nist.gov/vuln/detail/CVE-2026-56425))
- **Grafana's coordinated release fixes a CVSS 9.6 Snowflake data-source flaw plus a Loki path-traversal.** In **CVE-2026-28381 (CVSS 9.6)**, Grafana's Snowflake *data-source* plugin (the connector that lets Grafana query the Snowflake cloud data warehouse) exposed GET/PUT file commands, so any user able to run queries against it could read or write files between the Grafana server and the Snowflake host — turning a dashboard query into arbitrary file transfer. The same release fixes **CVE-2026-42129 (CVSS 7.7)**, a path traversal (escaping an intended directory via `../` sequences) in the Loki data-source plugin that an authenticated Viewer-role user could abuse to break out of the plugin's resource sandbox, and a geomap-panel stored-XSS (cross-site scripting — injecting attacker JavaScript that runs in another user's browser). Why it matters: Grafana sits at the centre of a lot of monitoring stacks with broad backend access, so a file-read/write primitive reachable by ordinary query users is a meaningful lateral-movement foothold. ([NVD CVE-2026-28381](https://nvd.nist.gov/vuln/detail/CVE-2026-28381); [Grafana advisory](https://grafana.com/security/security-advisories/cve-2026-28381))
- **Angular ships a multi-CVE security release across the framework and its VS Code Language Service.** Google's Angular web framework patched a cluster of high-severity issues in versions before 22.0.1 / 21.2.17 — **CVE-2026-54264, -54266, -54267, -54268** (CVSS 8.2–8.8) — alongside two flaws in the Angular Language Service VS Code extension, **CVE-2026-49241** and **CVE-2026-50178** (both CVSS 8.7), which gives template editing in the editor. Why it matters: client-side framework and developer-tooling bugs are easy to under-prioritise because they're not server CVEs, but the editor-extension flaws in particular put the developer's own machine in scope — update Angular and the extension together. ([NVD CVE-2026-54266](https://nvd.nist.gov/vuln/detail/CVE-2026-54266); [NVD CVE-2026-49241](https://nvd.nist.gov/vuln/detail/CVE-2026-49241))
- **CVE-2026-11746 — Central Dogma config server can fall back to a no-auth replication mode (CVSS 9.4).** In Line/LY's Central Dogma (an open-source git-backed configuration service) before 0.84.0, enabling ZooKeeper replication (ZooKeeper coordinates the cluster) without setting `replication.secret` leaves the replication channel unauthenticated — a misconfiguration the software permits rather than rejects, so a cluster can silently run wide open. A second issue, **CVE-2026-11745 (CVSS 8.8)**, is the git-mirror SSH client failing to verify remote host keys (no protection against a man-in-the-middle on the mirror connection). Why it matters: config servers hold the settings that drive everything downstream, so an unauthenticated replication path is a high-value target — set the secret and upgrade to 0.84.0. ([NVD CVE-2026-11746](https://nvd.nist.gov/vuln/detail/CVE-2026-11746); [NVD CVE-2026-11745](https://nvd.nist.gov/vuln/detail/CVE-2026-11745))
