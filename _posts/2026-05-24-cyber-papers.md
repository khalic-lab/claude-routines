## 🛡️ Cybersecurity

- **CVE-2026-20182 — Cisco Catalyst SD-WAN, CVSS 10.0, active exploitation by UAT-8616; federal patch deadline May 17 (passed).** A maximum-severity authentication bypass (CWE-287) in Cisco Catalyst SD-WAN Controller and SD-WAN Manager allows an unauthenticated remote attacker to send a single crafted CHALLENGE_ACK message over UDP port 12346 (DTLS). The vulnerability exploits a device-type-specific code path: when a connecting peer claims to be a vHub, certificate verification is skipped but the peer is still marked authenticated, granting full SD-WAN control-plane access. Cisco Talos attributed limited active exploitation with high confidence to UAT-8616, which used the foothold to add SSH keys, modify NETCONF configurations, and attempt root privilege escalation — effectively enabling fabric-wide network manipulation. Fixed in releases 20.9.9.1, 20.12.7.1, 26.1.1.1; no workarounds exist. CISA added to KEV with a federal deadline of May 17, 2026 (now passed). [via snippet] — [_The Hacker News_, May 2026](https://thehackernews.com/2026/05/cisa-adds-cisco-sd-wan-cve-2026-20182.html) / [NVD entry](https://nvd.nist.gov/vuln/detail/CVE-2026-20182)

- **CVE-2026-42897 — Microsoft Exchange OWA stored XSS, CVSS 8.1, active exploitation; federal deadline May 29.** Disclosed May 14, CVE-2026-42897 is a stored cross-site scripting vulnerability in Exchange Server Outlook Web Access affecting on-premises versions: 2016, 2019, and Subscription Edition (Exchange Online is not affected). An attacker sends a crafted email embedding obfuscated JavaScript; when the recipient opens the message in OWA, the browser executes the payload in the context of the authenticated session, enabling session hijacking and credential theft. Microsoft confirmed active exploitation but has not attributed it or published scale metrics. CISA added to KEV on May 15 with a federal remediation deadline of May 29, 2026 — five days away. Microsoft deployed an automatic mitigation (M2.1.x) via the Exchange Emergency Mitigation Service (EEMS) that is enabled by default on affected servers. [via snippet] — [_The Hacker News_, May 2026](https://thehackernews.com/2026/05/on-prem-microsoft-exchange-server-cve.html) / [_Microsoft TechCommunity_, May 2026](https://techcommunity.microsoft.com/blog/exchange/addressing-exchange-server-may-2026-vulnerability-cve-2026-42897/4518498)

- **Laravel-Lang supply chain: 700+ poisoned Composer versions exfiltrate cloud keys, CI tokens, crypto wallets.** Detected May 22, 2026, by Aikido Security: attackers who had gained write access to the laravel-lang GitHub organisation rewrote every git tag across three widely used Composer localisation packages (laravel-lang/http-statuses, laravel-lang/actions, laravel-lang/attributes) within a 15-minute window, redirecting tags to commits in an attacker-controlled fork. Because Packagist resolves release tags from GitHub, all 700+ affected versions silently delivered a cross-platform second-stage payload via an injected helpers.php file wired into composer's autoload.files — executed automatically on every PHP request immediately after `composer install` or `composer update`. The stager downloads a credential-harvesting binary that scrapes cloud provider keys (AWS, GCP, Azure), Kubernetes and Vault secrets, CI/CD tokens (GitHub Actions, GitLab CI, CircleCI), SSH private keys, .env files, browser-saved passwords, password-manager vaults, cryptocurrency wallet seeds, and messaging application tokens. Packagist unlisted the malicious versions within hours of disclosure. Operators who installed any affected package between approximately 22 May (17:00 UTC) and takedown should rotate all credentials in scope. [via snippet] — [_Aikido_, May 2026](https://www.aikido.dev/blog/supply-chain-attack-targets-laravel-lang-packages-with-credential-stealer) / [_BleepingComputer_, May 2026](https://www.bleepingcomputer.com/news/security/laravel-lang-packages-hijacked-to-deploy-credential-stealing-malware/)

- **CISA KEV May 21: Langflow CVE-2025-34291 (CVSS 9.4) and Trend Micro Apex One CVE-2026-34926 (CVSS 6.7); federal deadline June 4.** CISA added two actively exploited flaws to the KEV catalog on May 21, 2026. **CVE-2025-34291** is an origin validation error in Langflow, the open-source AI agent workflow builder: attackers combine permissive CORS (no origin check), absent CSRF protection, and a code-execution-by-design endpoint to achieve unauthenticated remote code execution and full system compromise without any prior authentication. **CVE-2026-34926** is a directory traversal in Trend Micro Apex One (on-premises) enabling a pre-authenticated local attacker to modify a key server table and inject arbitrary code that Apex One's agent-update mechanism then propagates to all managed endpoints — a CVSS 6.7 score that understates the blast radius given its lateral propagation potential. Federal agencies must remediate both under BOD 22-01 by June 4, 2026. [via snippet] — [_The Hacker News_, 21 May 2026](https://thehackernews.com/2026/05/cisa-adds-exploited-langflow-and-trend.html) / [CISA KEV catalog](https://www.cisa.gov/known-exploited-vulnerabilities-catalog)
