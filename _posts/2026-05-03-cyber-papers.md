## 🛡️ Cybersecurity — full day

- **CVE-2026-41940 — cPanel/WHM authentication bypass (CVSS 9.8), FCEB patch deadline TODAY; "Sorry" ransomware mass campaign** — A CRLF injection in cPanel's login and session-loading code allows attackers to inject arbitrary session-file properties (e.g. `user=root`) via a crafted `Authorization` header, granting unauthenticated administrative access. Actively exploited as a zero-day from at least late February; cPanel published advisory CPANEL-45341 and patches on **April 28**. CISA added CVE-2026-41940 to the Known Exploited Vulnerabilities catalog with a Federal Civilian Executive Branch (FCEB) remediation deadline of **May 3, 2026 — today**. Scale: Shodan shows ~1.5 M internet-exposed cPanel instances; at least 44,000 IP addresses confirmed compromised. Active campaign deploys **"Sorry" ransomware** — a Go-based Linux encryptor appending `.sorry`, using ChaCha20 with an embedded RSA-2048 public key (prevents decryption without attacker's private key), C2 via qTox P2P (no central server). Patch target: cPanel & WHM ≥ 120.0.15 / ≥ 118.0.41. `[active exploitation]` — [CVE-2026-41940 Detail, NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-41940) [via snippet]; [Critical cPanel flaw mass-exploited in "Sorry" ransomware attacks, BleepingComputer, May 2026](https://www.bleepingcomputer.com/news/security/critrical-cpanel-flaw-mass-exploited-in-sorry-ransomware-attacks/) [via snippet]; [CVE-2026-41940: cPanel & WHM Authentication Bypass, Rapid7, Apr 2026](https://www.rapid7.com/blog/post/etr-cve-2026-41940-cpanel-whm-authentication-bypass/) [via snippet]; [cPanel zero-day exploited for months before patch, Help Net Security, 30 Apr 2026](https://www.helpnetsecurity.com/2026/04/30/cpanel-zero-day-vulnerability-cve-2026-41940-exploited/) [via snippet]

- **CVE-2026-31431 "Copy Fail" — Linux kernel local privilege escalation (CVSS 7.8), CISA KEV May 1; exploitation expected to escalate** — Logic flaw in the `algif_aead` module of the AF_ALG (userspace crypto API) subsystem: an unprivileged user can abuse the interaction between AF_ALG sockets and the `splice()` syscall to achieve a **controlled 4-byte write into the kernel page cache**, enabling corruption of setuid binaries and root escalation. Affects virtually all distributions running kernels released since 2017, including Ubuntu 24.04 LTS, Amazon Linux 2023, RHEL 10.1, SUSE Linux 16, Debian, Fedora, and Arch Linux — impacting a substantial fraction of cloud and Kubernetes workloads. CISA added to KEV on **May 1, 2026**. Microsoft characterises current exploitation as "limited" (primarily PoC activity) but warns of "increased threat actor exploitation over the next few days." Vendor kernel updates address the flaw; apply via distribution package manager. `[active exploitation — limited, escalation anticipated]` — [CVE-2026-31431: Copy Fail vulnerability enables Linux root privilege escalation across cloud environments, Microsoft Security Blog, 1 May 2026](https://www.microsoft.com/en-us/security/blog/2026/05/01/cve-2026-31431-copy-fail-vulnerability-enables-linux-root-privilege-escalation/)

- **CVE-2026-40372 — ASP.NET Core Data Protection privilege escalation (CVSS 9.1), out-of-band patch April 21; key-ring rotation required post-upgrade** — A regression introduced in the .NET 10.0.6 April Patch Tuesday update broke HMAC validation in `Microsoft.AspNetCore.DataProtection`: the authentication code could calculate the HMAC over incorrect data or discard it entirely, breaking integrity guarantees. An unauthenticated network attacker can forge authentication cookies passing validity checks, potentially escalating to SYSTEM privileges. Affects `Microsoft.AspNetCore.DataProtection` versions 10.0.0–10.0.6. Microsoft issued an **emergency out-of-band patch on April 21** (version 10.0.7). **Critical post-patch step:** upgrading to 10.0.7 alone is insufficient — the DataProtection **key ring must be rotated** and all tokens/cookies issued during the vulnerable window (April 14–21) should be invalidated. No confirmed public exploitation at this time. — [Microsoft Patches Critical ASP.NET Core CVE-2026-40372 Privilege Escalation Bug, The Hacker News, 22 Apr 2026](https://thehackernews.com/2026/04/microsoft-patches-critical-aspnet-core.html) [via snippet]; [CVE-2026-40372 Detail, NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-40372) [via snippet]; [Microsoft out-of-band updates fix critical ASP.NET Core flaw, Security Affairs, Apr 2026](https://securityaffairs.com/191130/security/microsoft-out-of-band-updates-fixed-critical-asp-net-core-privilege-escalation-flaw.html) [via snippet]
