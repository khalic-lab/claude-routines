## 🛡️ Cybersecurity

- **Splunk Enterprise unauthenticated file-write flaw added to CISA's KEV — patch by Saturday (CVE-2026-20253).** CISA added CVE-2026-20253 to its Known Exploited Vulnerabilities catalog (KEV — the US cyber agency's list of bugs confirmed to be actively attacked in the wild) on 2026-06-18, with a federal remediation deadline of 2026-06-21. The flaw is a *missing authentication for a critical function*: an unauthenticated user can create or truncate (zero out) arbitrary files through a PostgreSQL sidecar service endpoint in Splunk Enterprise. Splunk is the log-and-SIEM backbone (security information and event management — the system that collects and alerts on security logs) in many enterprises, so an unauthenticated file-write on the box that *watches* everything else is a serious foothold; the three-day KEV deadline signals real exploitation pressure. ([NVD, CVE-2026-20253](https://nvd.nist.gov/vuln/detail/CVE-2026-20253); [CISA KEV catalog](https://www.cisa.gov/known-exploited-vulnerabilities-catalog))

- **Two auth-bypass bugs in Google's MCP Toolbox let attackers slip past opaque-token checks (CVE-2026-11717 / CVE-2026-11718, CVSS 9.3).** Both flaws sit in the `validateOpaqueToken` path of `googleapis/mcp-toolbox` — Google's server-side toolkit for exposing databases to AI agents over MCP (Model Context Protocol, the now-standard way LLM agents call external tools). When the toolbox validates an opaque token via an OAuth 2.0 introspection endpoint (RFC 7662 — the spec where a server asks the issuer "is this token still active?"), it decodes the response into a struct whose `Active` field is a pointer, and the downstream claim-checking logic mishandles the unset/nil case — so a crafted or empty introspection response can be treated as a *valid, active* session. Net effect: an attacker bypasses authentication entirely (CVSS 9.3 — critical). It lands squarely in the fast-growing, lightly-audited MCP plumbing that's wiring LLMs to production data. ([NVD, CVE-2026-11717](https://nvd.nist.gov/vuln/detail/CVE-2026-11717); [NVD, CVE-2026-11718](https://nvd.nist.gov/vuln/detail/CVE-2026-11718))

([NVD, CVE-2026-12569](https://nvd.nist.gov/vuln/detail/CVE-2026-12569))

## 📄 ML research — second arXiv batch

_All five papers below were submitted to arXiv on 2026-06-17 (the current US-time batch, which rolls over at 20:00 ET), verified via the Atom API, and none was cited in this morning's Overview brief._

- **Turing-RL — training user simulators with a "Turing-test" reward.** arXiv 2606.19336 (submitted 2026-06-17) `[preprint]`. Realistic simulated users would help train and evaluate AI assistants, but the usual recipe — make a language model match one ground-truth human reply, by maximising its probability or a similarity score — produces bland, averaged behaviour. Turing-RL instead trains the simulator with reinforcement learning against a *discriminative Turing reward*: an LLM judge tries to tell the simulator's output from a real human's, and the simulator is rewarded for being indistinguishable. Why it matters: a more faithful synthetic-user generator improves how we train and stress-test conversational agents, and the discriminator framing sidesteps the "regression to the mean" problem of imitation losses. ([abstract](https://arxiv.org/abs/2606.19336))

- **UBP2 — uncertainty-balanced exploration for preference-based RL.** arXiv 2606.19328 (submitted 2026-06-17) `[preprint]`. Preference-based reinforcement learning learns a reward model from pairwise "A is better than B" comparisons, dodging the need to hand-design a reward function — but it usually collects data passively and is sample-hungry early on. UBP2 makes the agent *plan* its exploration, jointly reasoning over uncertainty in the reward, the environment dynamics, and the value function (using ensembles of each) to seek out the situations where a human comparison would teach it the most. Why it matters: cutting the number of human preference labels needed is exactly the bottleneck in aligning agents to messy, hard-to-specify goals. ([abstract](https://arxiv.org/abs/2606.19328))

- **Large Language Gibbs — coherent structured inference from an LLM's own conditionals.** arXiv 2606.19264 (submitted 2026-06-17) `[preprint]`. LLMs encode a lot of world knowledge, but extracting it in a *probabilistically coherent* way over many interrelated variables is hard: a single left-to-right pass commits to early choices it can't revisit. This work uses the model's next-token conditional distributions as the transition operators of a Gibbs sampler (a classic statistics method that draws a joint sample by repeatedly resampling one variable given all the others), iteratively re-drawing each variable conditioned on the rest. Why it matters: it turns an LLM into a usable engine for structured probabilistic reasoning — useful wherever you need consistent joint estimates, not just a fluent single answer. ([abstract](https://arxiv.org/abs/2606.19264))

- **OmniAgent — active perception as a POMDP for efficient long-video understanding.** arXiv 2606.19341 (submitted 2026-06-17) `[preprint]`. "Watch every frame" models pay a compute cost that grows with video length regardless of how hard the question is, and even interactive systems often pre-scan the whole clip. OmniAgent frames the task as a POMDP (partially observable Markov decision process — sequential decisions made with incomplete information) and runs an observe→think→act loop, taking on-demand actions to pull just the audio-visual cues it needs into a persistent text memory. Why it matters: it decouples cost from video duration and ties it to *query difficulty* instead — a route to making hour-long video Q&A affordable. ([abstract](https://arxiv.org/abs/2606.19341))

- **NeSyCat Torch — one differentiable framework for all of neurosymbolic learning.** arXiv 2606.19279 (submitted 2026-06-17) `[preprint]`. Neurosymbolic AI tries to fuse neural networks with logical/symbolic reasoning, but the field is fragmented: classical, fuzzy, probabilistic and neural systems each define "truth" by their own rules, so methods don't compose. NeSyCat (extending the ULLER framework) subsumes them under a single definition of truth parameterised by a *monad* and an aggregation structure — the categorical-semantics machinery that lets you swap "which logic" the way you swap a hyperparameter — and NeSyCat Torch supplies the missing piece: interpreting learned predicates and functions via neural nets, implemented on a differentiable tensor backend. Why it matters: a common, trainable substrate could let neurosymbolic methods be mixed and benchmarked head-to-head instead of reinvented per paradigm. ([abstract](https://arxiv.org/abs/2606.19279))
