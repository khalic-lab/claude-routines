## 🛡️ Cybersecurity

- **FreeBSD discloses a batch of kernel local-privilege-escalation flaws.** A coordinated set of FreeBSD kernel advisories landed in NVD today, several allowing an unprivileged local user to escalate to root. Highlights: **CVE-2026-49412** — a use-after-free (UAF — memory is freed then accessed again, letting an attacker control freed memory) in the IPv6 `IPV6_MSFILTER` handler, which drops and re-acquires a lock while copying a source-filter list from userspace, opening a race window; **CVE-2026-49417** — a UAF in the audio (`/dev/dsp`) device, world-accessible by default, where the audio buffer can be freed while a memory mapping to it remains valid; **CVE-2026-49416** — an integer overflow in the console `CONS_HISTORY` ioctl that under-allocates a heap buffer and then writes past it (a classic heap overflow); and **CVE-2026-45258** — an integer-overflow bounds-check bypass in `dsp_mmap_single()` where offset+length wraps around to pass validation. NVD has not yet assigned CVSS scores, but the class (local root on default-accessible device nodes) is serious for multi-user and hosting systems; FreeBSD has published patches. ([NVD: CVE-2026-49412](https://nvd.nist.gov/vuln/detail/CVE-2026-49412); see also [CVE-2026-49417](https://nvd.nist.gov/vuln/detail/CVE-2026-49417), [CVE-2026-49416](https://nvd.nist.gov/vuln/detail/CVE-2026-49416), [CVE-2026-45258](https://nvd.nist.gov/vuln/detail/CVE-2026-45258))

- **CVE-2026-12415 — unauthenticated account takeover in the WordPress "Invoice Generator" plugin, CVSS 9.8 (critical).** A missing capability check on the `pravel_invoice_edit_account()` AJAX action — exposed via the unauthenticated `wp_ajax_nopriv_*` hook in versions up to and including 1.0.0 — lets any unauthenticated visitor supply an attacker-controlled `user_id` and `user_email` and rewrite another account's details, enabling privilege escalation/takeover. CVSS 9.8 means trivially exploitable over the network with no authentication and full impact. This is the same failure class (unauthenticated identity-validation gaps in WordPress plugins) that produced the Branda 9.8 account-takeover a week ago — a *distinct* plugin and CVE, but the recurring pattern is worth noting for anyone running WordPress: audit `nopriv` AJAX handlers. Update or remove the plugin. ([NVD: CVE-2026-12415](https://nvd.nist.gov/vuln/detail/CVE-2026-12415))

- **A "clean" GitHub repo can trick AI coding agents into running malware — no exploit code, nothing to approve.** Researchers at Mozilla's 0DIN (Zero Day Investigative Network) demonstrated that an agentic coding tool asked to clone and set up a benign-looking GitHub repository can be steered into executing a malicious payload that stays invisible to security scanners, the AI agent, and human reviewers — "no exploit code, no warning, no suspicious command anyone had to approve." Their proof-of-concept used Claude Code to run a cloned project and ended with an interactive reverse shell on the developer's machine, with the malicious behaviour smuggled through setup/config steps rather than checked-in code. The takeaway for anyone using AI dev agents (this reader included): treat "clone and run this repo" the way you'd treat piping a stranger's script into your shell — agent autonomy collapses the human review step that normally catches this. ([BleepingComputer, 27 Jun 2026](https://www.bleepingcomputer.com/news/security/clean-github-repo-tricks-ai-coding-agents-into-running-malware/))

- **Mandiant details how Cisco Catalyst SD-WAN flaws were exploited as zero-days for root access.** Mandiant published an analysis of in-the-wild attacks chaining Cisco Catalyst SD-WAN (the centrally-managed wide-area-network fabric, formerly vManage/vSmart/vBond) vulnerabilities to gain root on controllers. The anchor bug, **CVE-2026-20127** (CVSS 10.0 — maximum), is a peering-authentication flaw in the SD-WAN Controller/Manager/Validator; a related **CVE-2026-20182** (CVSS 10.0) was found and fixed after the February disclosure, and **CVE-2026-20245** (CVSS 7.8) is a CLI flaw. Root on an SD-WAN controller means control of the entire managed network's routing — patch and hunt for compromise on internet-exposed management planes. ([NVD: CVE-2026-20127](https://nvd.nist.gov/vuln/detail/CVE-2026-20127); [BleepingComputer/Mandiant, 24 Jun 2026](https://www.bleepingcomputer.com/news/security/mandiant-reveals-how-cisco-sd-wan-zero-day-attacks-gained-root-access/))
