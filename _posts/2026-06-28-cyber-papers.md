## 🛡️ Cybersecurity

A quiet weekend for vendor advisories but a busy one for independent disclosure: a single researcher (publishing via the "exploitarium" repo and VulnCheck advisories) dropped coordinated proof-of-concepts for several of today's higher-severity bugs (Gitea, FFmpeg, RustDesk). NVD published 36 CVEs dated today; the notable ones below.

- **CVE-2026-58053 — Gitea `act_runner` Docker container-hardening bypass (CVSS 9.9 v3.1 / 9.4 v4.0; PoC public).** Gitea's `act_runner` (the agent that executes CI/CD workflows) with the Docker backend passes a workflow's `container.options` string straight into the Docker container's `HostConfig`. When configured with `privileged: false`, it only forces the *Privileged* flag off while leaving dangerous options like `--pid=host`, `--cap-add` and `--security-opt` untouched — so anyone who can run a workflow can spin up a job container that escapes its sandbox (container escape: breaking out of the isolated container to the host). Affects `act` through 0.262.0. Why it matters: critical (CVSS 9.9 — trivially reachable by any user who can submit a workflow, full host compromise) for anyone self-hosting Gitea CI on a Docker runner; treat untrusted workflow submitters as having root on the runner host until patched. ([NVD CVE-2026-58053](https://nvd.nist.gov/vuln/detail/CVE-2026-58053); [VulnCheck advisory](https://www.vulncheck.com/advisories/gitea-act-runner-container-hardening-bypass-via-workflow-container-options))

- **CVE-2026-10643 — Zephyr RTOS `recvmsg()` out-of-bounds write (CVSS 8.7).** In the Zephyr real-time OS (a widely used open-source RTOS for microcontrollers and IoT), the IP-socket `recvmsg()` implementation validated the user-supplied ancillary buffer (`msg_control` — the side-channel buffer that carries control metadata alongside a received packet) using only the payload length, omitting the size of the control-message header. A crafted control buffer length therefore lets an attacker write past the allocation — an out-of-bounds write (memory corruption beyond an allocated region). Fixed in a committed patch. Why it matters: high severity on embedded/IoT fleets where Zephyr is the firmware base and field updates are slow; corruption of network-stack memory is a classic path to device takeover. ([NVD CVE-2026-10643](https://nvd.nist.gov/vuln/detail/CVE-2026-10643)) A companion Zephyr socket bug, **CVE-2026-10646** (`getaddrinfo()`, CVSS 7.4), was published the same day. ([NVD CVE-2026-10646](https://nvd.nist.gov/vuln/detail/CVE-2026-10646))

- **CVE-2026-58049 — FFmpeg RASC decoder out-of-bounds write (CVSS 8.6 v3.1 / 8.8 v4.0; PoC public).** FFmpeg's RASC video decoder (`decode_dlta` in `libavcodec/rasc.c`) does 32-bit reads/writes at the row cursor *before* the row-boundary check and validates the delta region in pixel rather than byte units, so a crafted clip using the RASC codec can write several bytes past the row allocation. This is the second FFmpeg decoder out-of-bounds-write disclosed this month — a *distinct* bug from the mid-June MagicYUV flaw (CVE-2026-8461), in a different decoder. Why it matters: FFmpeg is embedded in countless media pipelines and servers, so a bitstream-controlled memory write means a malicious media file can corrupt any service that auto-transcodes uploads. ([NVD CVE-2026-58049](https://nvd.nist.gov/vuln/detail/CVE-2026-58049); [VulnCheck advisory](https://www.vulncheck.com/advisories/ffmpeg-out-of-bounds-write-in-rasc-decoder-decode-dlta))

- **CVE-2026-8095 — WordPress "Frontend File Manager" plugin: authenticated arbitrary file deletion (CVSS 8.1).** In versions ≤ 23.6, a case-sensitivity bug lets a logged-in user bypass parameter sanitisation (supplying `WPFM_DIR_PATH` in uppercase evades an `unset` check, then gets normalised back to the real parameter) to delete arbitrary files via the `wpfm_file_meta_update` AJAX handler. Deleting `wp-config.php` can knock a site into a re-installation state attackers can hijack. Why it matters: low-privilege users (subscribers, contributors) on any of the plugin's installs can wipe site files — patch or remove the plugin. ([NVD CVE-2026-8095](https://nvd.nist.gov/vuln/detail/CVE-2026-8095); [Wordfence](https://www.wordfence.com/threat-intel/vulnerabilities/id/506006ce-7b1c-4f9d-93f3-abc87abea2bb?source=cve))

- **CVE-2026-58056 — RustDesk file-transfer session can inject keyboard/mouse and capture the screen (CVSS 7.6; PoC public).** RustDesk (a popular open-source remote-desktop tool) gates incoming control messages on per-capability flags rather than on the session's authorised connection type, and a file-transfer session never clears those flags. So a peer holding only a *FileTransfer* authorisation can inject keyboard and mouse input and reach screenshot/display-capture handlers — acting well outside its granted scope. Why it matters: a help-desk or vendor given "just file transfer" access effectively gets full interactive control; an authorisation-scope bypass like this undermines the whole least-privilege model of granular remote-access grants. ([NVD CVE-2026-58056](https://nvd.nist.gov/vuln/detail/CVE-2026-58056); [VulnCheck advisory](https://www.vulncheck.com/advisories/rustdesk-filetransfer-session-authorization-scope-bypass))

Why it matters: KEV listing means real-world exploitation is observed, not theoretical — these jump the patch queue, especially the network-edge Cisco and Ubiquiti gear. ([CISA KEV — CVE-2026-20230](https://nvd.nist.gov/vuln/detail/CVE-2026-20230); [CVE-2026-12569](https://nvd.nist.gov/vuln/detail/CVE-2026-12569); [CVE-2026-34909](https://nvd.nist.gov/vuln/detail/CVE-2026-34909))
