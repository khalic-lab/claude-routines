## 🛡️ Cybersecurity

- **CISA adds two unauthenticated server-side flaws to the KEV catalog (added 2026-06-25), with a 2026-06-28 federal patch deadline.** The KEV ("Known Exploited Vulnerabilities" — CISA's list of flaws confirmed exploited in the wild, which US agencies must patch by the due date) gained **Cisco Unified Communications Manager SSRF — CVE-2026-20230**. SSRF (server-side request forgery — tricking a server into making attacker-chosen network requests) here lets an *unauthenticated* remote attacker write files onto the underlying OS in Cisco's enterprise voice/telephony platform, a foothold that can later be escalated to root. The same KEV batch re-listed the PTC Windchill/FlexPLM unauth RCE (CVE-2026-12569, disclosed and covered here 2026-06-18) — its KEV addition now makes patching it federally mandatory. Both due 2026-06-28; neither yet tied to a named ransomware campaign. [NVD: CVE-2026-20230](https://nvd.nist.gov/vuln/detail/CVE-2026-20230) · [CISA KEV catalog](https://www.cisa.gov/known-exploited-vulnerabilities-catalog)
- **GeoVision license-plate cameras: a cluster of unauthenticated CVSS 9.8 RCEs — and the hardware is end-of-life.** GeoVision **GV-LPC2011 / GV-LPC2211** (V1.12 and earlier) license-plate-capture cameras carry a swarm of unauthenticated network-reachable memory-corruption bugs across multiple services — stack buffer overflows in `vlsvr`, `ssvr` and `thttpd` (**CVE-2026-57878/57879/57880/57881**, all **9.8 critical** — i.e. trivially exploitable, full device takeover), plus an OOB write, format-string, NULL-deref and 802.1x-upload overflows (CVE-2026-57873/74/75/76/77). These are internet-facing roadside/parking devices that often sit on flat networks; treat any exposed unit as compromisable and segment or retire it. [NVD: CVE-2026-57881](https://nvd.nist.gov/vuln/detail/CVE-2026-57881)
- **Node.js ships a TLS hostname-verification bypass affecting every supported line — CVE-2026-48618 (7.7 high).** A Unicode dot-separator normalisation mismatch between Node's hostname *resolver* and its certificate *verifier* lets a wildcard certificate (e.g. `*.example.com`) authenticate hostnames it should not — a "wildcard-depth" bypass that can break the security boundary TLS is supposed to enforce (a server you connect to may not be the one whose name the cert vouches for). It affects **Node.js 22, 24 and 26**; upgrade to the patched releases. Relevant to anyone running Node services behind hostname-based trust. [NVD: CVE-2026-48618](https://nvd.nist.gov/vuln/detail/CVE-2026-48618)
- **KubeVirt cross-tenant VM takeover when live-migration TLS is disabled — CVE-2026-13325 (8.5 high).** KubeVirt runs virtual machines inside Kubernetes. With `migrations.disableTLS: true`, the target `virt-handler` opens a plain, unauthenticated TCP listener bound to *all* interfaces (0.0.0.0) that proxies straight into the VM's `virtqemud` control socket — so any pod on the cluster network can issue raw libvirt commands against *another tenant's* VM: read its memory, change its state, or destroy it. The sting in the tail: setting a dedicated migration network does **not** move the bind address, and the docs never disclosed that disabling TLS also removes all authentication. Audit for `disableTLS` in multi-tenant clusters. [NVD: CVE-2026-13325](https://nvd.nist.gov/vuln/detail/CVE-2026-13325)
- **Canonical LXD guest can mount and overwrite another guest's storage volume — CVE-2026-12411 (8.4 high).** A broken-access-control flaw in LXD's `devLXD` device-patch handler lets an untrusted guest craft a device `PATCH` request over `/dev/lxd` to mount, read and overwrite a *different* guest's custom storage volume, when `security.devlxd.management.volumes` is enabled — a container-isolation break of the kind that turns a single tenant foothold into cross-tenant data access. [NVD: CVE-2026-12411](https://nvd.nist.gov/vuln/detail/CVE-2026-12411)
- **Even the pentester's tool is vulnerable: insecure deserialization in OWASP ZAP's ViewState add-on — CVE-2026-57527 (8.7 high).** A malicious proxied web server can embed a crafted serialized Java object in the `javax.faces.ViewState` response field; ZAP's add-on base64-decodes it and hands it straight to `ObjectInputStream.readObject()` with no allowlist or type filter — the classic "deserialization gadget" that runs attacker code (here, inside ZAP's own JVM) the moment the analyst renders the ViewState panel. Fixed in ViewState add-on v4. A neat reminder that security tooling that parses hostile input is itself attack surface. [NVD: CVE-2026-57527](https://nvd.nist.gov/vuln/detail/CVE-2026-57527)

## 📄 ML research — second arXiv batch

_Second submission window of 2026-06-25 (arXiv batches roll over at 20:00 ET; today's 06-26 batch had not yet published at compose time). IDs below are distinct from this morning's Overview batch._

- **arXiv:2606.27288 (submitted 2026-06-25) — there's a hard ceiling on how much routing/voting/mixture-of-agents can beat a single model.** [preprint] Combining LLMs (routing, voting, cascades, mixture-of-agents) is supposed to outscore any one model. The authors prove that for any policy whose answer is *one member model's* answer, accuracy can't exceed `1 − β`, where **β is the rate at which *every* model is wrong on the same query** (the "co-failure" floor). Crucially, the metric the field usually reports — average pairwise error correlation ρ — *can't* recover β: ensembles with identical correlations can have very different all-wrong rates. Tested across 67 frontier models, with a finite-sample (Clopper–Pearson) certificate on the best gain any router could deliver. Why it matters: a sober counterweight to this morning's "Capability Frontier" paper — it bounds how far clever orchestration can take you before you simply need a better base model. [abstract](https://arxiv.org/abs/2606.27288)
- **arXiv:2606.27326 (submitted 2026-06-25) — hallucination in generative world models is predictable, and you can head it off.** [preprint] A "world model" learns to roll forward realistic, action-controllable video of an environment (useful for planning and game/robot simulators), but its rollouts stay visually fluent while quietly drifting from real dynamics. The authors show this drift concentrates in **low-coverage regions of the state–action space** — places the training data barely visited — so cheap data-centric signals can both *detect* and *steer away from* it. They release **MMBench2** (427 hours, 210 tasks, with ground-truth actions/rewards and live simulators) and name three failure modes (perceptual, action-marginalised, scene-diverging). Practical takeaway: hallucination here is a coverage problem, not an irreducible mystery. [abstract](https://arxiv.org/abs/2606.27326)
- **arXiv:2606.27229 (submitted 2026-06-25) — CARVE: gate the memory on the key axis only, and linear attention gets both cheaper and trainable in parallel.** [preprint] Recurrent sequence models (the efficient, sub-quadratic alternative to Transformer attention) "must forget to remember," but state-of-the-art gating decides what to erase looking only at the incoming token, blind to the memory it's modifying. CARVE argues the leading delta-rule design (GDN-2) has three coupled defects and fixes all three with one move — **erase only on the key axis, not the value axis**. The payoff is concrete: the value-axis mask was both wasting parameters and (they prove) mathematically blocking the chunk-parallel "WY-form" solver that makes recurrent training competitive with Transformers — so key-axis-only gating restores efficient parallel training. A genuinely architectural contribution to the efficient-attention line. [abstract](https://arxiv.org/abs/2606.27229)
- **arXiv:2606.27369 (submitted 2026-06-25) — RiVER: reinforcement learning for LLMs when you have no ground-truth answer.** [preprint] RLVR (reinforcement learning with verifiable rewards) normally needs the correct answer to score each attempt, which rules out tasks where no ground truth exists. RiVER ("Ranking-induced VERifiable") instead derives a *continuous* reward from deterministic execution feedback — how well an output scores on the task — and tackles two failure modes that wreck naïve continuous-reward RL: **scale dominance** (uncalibrated score magnitudes across instances distort the gradient) and **frequency dominance** (a few repeated instances drowning out the rest). Why it matters: it widens RL post-training to open-ended, "no answer key" objectives like optimisation and design. [abstract](https://arxiv.org/abs/2606.27369)
- **arXiv:2606.27199 (submitted 2026-06-25) — steering LLM forecasters away from cheating on the future.** [preprint] When you ask an LLM to forecast, does it reason from generalizable patterns or lean on **look-ahead bias** — leaking knowledge of events that, for the forecast date, hadn't happened yet? The authors use sparse autoencoders (SAEs — they decompose tangled activations into interpretable features) to locate both "time-aware" and "look-ahead-biased" features, then *intervene*: amplifying the time-awareness features substantially cuts look-ahead bias on forecasting prompts while preserving general reasoning. A clean interpretability-to-control story with an obvious use in backtest-honest evaluation. [abstract](https://arxiv.org/abs/2606.27199)
- **arXiv:2606.27359 (submitted 2026-06-25) — when is a more-likely answer actually a more-correct one?** [preprint] Most decoding strategies push probability mass toward higher-likelihood outputs (locally per token or globally per sequence), so they only work to the extent that **sequence probability tracks correctness** — and that link is rarely measured directly. This paper quantifies it across decoding methods, hyperparameters, prompt–answer pairs and repeated samples, mapping where "the likely answer is right" holds and where it breaks. Useful grounding for anyone tuning sampling/decoding and wondering why confident generations still go wrong. [abstract](https://arxiv.org/abs/2606.27359)
