- **A wave of critical Joomla extension bugs is now an active-exploitation story: the JCE editor flaw (CVE-2026-48907) was added to CISA's KEV catalog, and three fresh CVSS-10/9.5 siblings landed today.** KEV (Known Exploited Vulnerabilities) is CISA's list of bugs confirmed exploited in the wild — a KEV listing means "patch now, not next sprint." CVE-2026-48907 (Joomla Content Editor / JCE, improper access control) lets unauthenticated users upload and run PHP via crafted editor profiles, and CISA added it on 2026-06-16 with a federal remediation deadline. Published today in the same family: **CVE-2026-48908** (SP Page Builder by JoomShaper, **CVSS 10.0** — unauthenticated arbitrary file upload → PHP code execution, i.e. full server takeover with no login), **CVE-2026-48939** (iCagenda event extension, **CVSS 10.0**, arbitrary file upload via the attachment feature → RCE), and **CVE-2026-48909** (SP LMS / com_splms < 4.1.4, **CVSS 9.5** — an unauthenticated *deserialization* bug, where the app rebuilds an object from attacker-controlled cookie data and that reconstruction itself runs code). Why it matters: Joomla powers a large slice of small-business and association sites, these are pre-auth and trivially scriptable, and one of them is already being exploited — expect mass scanning. Patch/upgrade the affected extensions immediately. _[ongoing since 2026-06-05]_ ([NVD: CVE-2026-48908](https://nvd.nist.gov/vuln/detail/CVE-2026-48908); [NVD: CVE-2026-48907 (KEV)](https://nvd.nist.gov/vuln/detail/CVE-2026-48907); [NVD: CVE-2026-48939](https://nvd.nist.gov/vuln/detail/CVE-2026-48939))

- **AI/LLM tooling keeps shipping security debt: bugs disclosed today hit Flowise (an LLM app-builder) and — ironically — picklescan, the tool meant to *catch* malicious model files.** Three to know. **CVE-2024-58351** (Flowise < 2.1.4, **CVSS 9.3**): the `overrideConfig` option — on by default, exposed in both the web widget and the Prediction API with no allow-list — lets a caller inject configuration into the running chainflow, leading to remote code execution. **CVE-2026-56267** (Flowise < 3.0.13, CVSS 6.9): the unauthenticated `forgot-password` endpoint returns full user objects including PII, so an attacker can enumerate valid emails and harvest data. **CVE-2026-56304** (picklescan < 1.0.1, CVSS 6.9): an unsafe-pickle flaw — Python's `pickle` format can execute code on load, which is exactly the risk picklescan exists to flag — lets a crafted payload instantiate `logging.FileHandler` to create arbitrary zero-byte files during the scan itself, turning a defensive scanner into an attack surface. Why it matters to anyone building on LLM infrastructure: these are the supply-chain seams of the AI stack (agent builders, model-file scanners), and "the security tool has the vuln" is a recurring 2026 theme. Upgrade Flowise to ≥3.0.13 and picklescan to ≥1.0.1. ([NVD: CVE-2024-58351](https://nvd.nist.gov/vuln/detail/CVE-2024-58351); [NVD: CVE-2026-56267](https://nvd.nist.gov/vuln/detail/CVE-2026-56267); [NVD: CVE-2026-56304](https://nvd.nist.gov/vuln/detail/CVE-2026-56304))

The headline issues (all "before 12.128.2" unless noted): **CVE-2026-56216** (CVSS 8.7) — a *scope escalation* where an app-limited API key can mint an unrestricted key via `POST /functions/v1/apikey`; **CVE-2026-56215** (CVSS 8.7, fixed in 12.128.12) — authenticated users can change their `public.users.email` to an arbitrary address that the SSO provisioning endpoint then trusts (account-takeover primitive); **CVE-2026-56214** (CVSS 8.7) — unauthenticated info disclosure via Supabase PostgREST RPC endpoints (`is_trial_org`, `is_paying_org`). There's also **CVE-2026-56218** — uploaded images aren't stripped of EXIF metadata, leaking GPS geolocation. Why it matters: chain the scope-escalation and email-trust bugs and an attacker can move from a limited tenant to broader control of the update pipeline. Self-hosted Capgo operators should upgrade to ≥12.128.12. ([NVD: CVE-2026-56216](https://nvd.nist.gov/vuln/detail/CVE-2026-56216); [NVD: CVE-2026-56215](https://nvd.nist.gov/vuln/detail/CVE-2026-56215); [NVD: CVE-2026-56214](https://nvd.nist.gov/vuln/detail/CVE-2026-56214))
