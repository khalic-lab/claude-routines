## 🚀 New models, datasets, & open weights

The day's headline, and it resolves a thread this brief has been tracking since 1 June.

- **MiniMax M3 open weights finally land `[ongoing since 2026-06-01]` `[vendor PR]`.** The
  Shanghai lab **MiniMax** published the full open weights for **M3** on Hugging Face today,
  closing a release that was overdue at yesterday's 10-day mark. What's now actually on the
  Hub (a direct Hub API query confirms the files, not just a press release):
  - **Architecture:** `minimax_m3_vl` — a **mixture-of-experts** (MoE — only a fraction of
    the model's parameters fire per token, so a large model runs at a smaller model's compute
    cost), **~427B total parameters**, multimodal (image-text-to-text plus video input;
    text-only output). License is listed as `other` — i.e. a custom license, **not** a clean
    Apache/MIT open-source grant, so check the terms before commercial use.
  - **Context:** 1M tokens (one million — roughly a large codebase or a stack of documents in
    a single prompt), built on the lab's new sparse-attention method (see 📊).
  - **Positioning:** "agent / coding / video" per the model tags — MiniMax is aiming this at
    agentic coding (a model that reads a whole repo and edits across files) and computer-use,
    the same niche as DeepSeek V4 and Qwen3.7.
  - **Ecosystem on day one:** within hours, community ports appeared — an **unsloth GGUF**
    build (the format `llama.cpp` and friends use for local CPU/GPU inference), an official
    **MXFP8** build (8-bit floating-point quantization to shrink the memory footprint), and
    third-party NVFP4 (4-bit) quants. That breadth on launch day is itself a signal the
    weights are genuinely public and usable, not a token drop.

  _Why it matters:_ this is the first time an open-weight model has bundled frontier-tier
  coding claims, a 1M-token window, **and** native multimodality in one downloadable
  checkpoint — if the independent numbers hold (they're still settling; see 📊), it resets
  the ceiling for what you can self-host without paying a frontier API.
  [MiniMax-M3 model card, Hugging Face](https://hf.co/MiniMaxAI/MiniMax-M3) ·
  [MiniMax-M3-MXFP8 (official 8-bit)](https://hf.co/MiniMaxAI/MiniMax-M3-MXFP8) ·
  [the-decoder, 2026-06-04](https://the-decoder.com/minimax-m3-open-weight-model-with-a-million-token-context-challenges-proprietary-leaders/) `[via snippet]`

## 📊 Benchmarks & evaluations

The benchmark story is the *same* M3 vendor claims this brief flagged on 4 June — but with
one thing that's genuinely new today: **the weights shipped, so the numbers are now
independently testable** instead of being vendor slides. That reclassifies them from
"unverifiable" to "pending verification."

- **MiniMax M3 — vendor benchmark claims, now reproducible `[ongoing since 2026-06-04]`
  `[vendor PR]`.** The headline figures MiniMax has been advertising:
  - **SWE-Bench Pro 59.0%** — SWE-Bench is the standard "can the model fix a real GitHub
    issue end-to-end" test; the "Pro" split is the harder, contamination-resistant version,
    and ~59% would edge GPT-5.5 and Gemini 3.1 Pro *if* it replicates on independent harnesses.
  - **Terminal-Bench 2.1 66.0%**, **OSWorld-Verified 70.06%** (driving a real desktop GUI to
    complete tasks), **MCP Atlas 74.2%** (tool-use over the Model Context Protocol — how well
    the model orchestrates external tools/agents).

  These are still **vendor-run numbers**; the change today is that anyone can now download the
  weights and re-run them, which is exactly what was impossible last week. Treat the 59% as a
  claim awaiting a clean third-party reproduction, not a settled result.
  [the-decoder, 2026-06-04](https://the-decoder.com/minimax-m3-open-weight-model-with-a-million-token-context-challenges-proprietary-leaders/) `[via snippet]` `[single-source for the agentic numbers]`

- **Artificial Analysis Intelligence Index — M3 at 55 `[via snippet]`.** The independent
  aggregator Artificial Analysis (which runs its own harness across many evals and rolls them
  into one composite "Intelligence Index") scored M3 at **55**, and flagged that once the
  weights were public it would be the **highest-scoring open-weight model** on its GDPval-AA
  economic-task eval. AA's page 403s the sandbox so this is a search-excerpt citation — but
  it's a useful independent counterweight to MiniMax's own slides, and it's the number to
  watch as the post-release re-runs come in.
  [Artificial Analysis — MiniMax M3](https://artificialanalysis.ai/articles/minimax-m3) `[via snippet]`

- **The method behind the long context — MiniMax Sparse Attention (MSA) `[preprint]`.** The
  technical report accompanying the weights (arXiv 2606.13392) describes **MSA**, a *blockwise
  sparse attention* scheme — instead of every token attending to every other token (the
  quadratic cost that makes long context expensive), it routes attention through a coarse
  "index branch" that picks which blocks matter, then a "main branch" that attends only
  within them, on top of grouped-query attention. The reported efficiency wins are the real
  contribution: **~28.4× less per-token attention compute at 1M context**, and **14.2× faster
  prefill / 7.6× faster decoding** wall-clock on H800 GPUs. _Why it matters:_ cheap, accurate
  1M-token attention is the bottleneck for whole-repo and whole-corpus agents; if MSA holds up
  outside the authors' setup, it's a reusable recipe, not just one model's trick. (One honest
  caveat: the abstract benchmarks the method on a 109B-parameter configuration, while the
  shipped M3 checkpoint is ~427B total — the paper is the *method*, so don't read its ablation
  sizes as the product's.) [arXiv 2606.13392 (abstract)](https://export.arxiv.org/abs/2606.13392)

## 🍎 Apple Silicon / on-device

Unusually, this section isn't empty — courtesy of M3's launch-day port wave.

- **MiniMax M3 GGUF + quantized ports, day one.** An **unsloth GGUF** build (the format
  `llama.cpp` / LM Studio / Ollama load) and an official **MXFP8** 8-bit build appeared on the
  Hub the same day as the weights, alongside third-party NVFP4 4-bit quants. _The honest
  caveat for Apple-Silicon readers:_ M3 is a **~427B-parameter** model — even at 4-bit that's
  on the order of ~200GB+ of weights, so this is **Mac Studio / multi-GPU territory, not a
  MacBook**. The MoE design means only a slice activates per token (so it *runs* faster than
  its size suggests once loaded), but you still have to *hold* it in memory. A first-class
  **MLX** (Apple's array framework for Apple-Silicon inference) port wasn't visible in the
  Hub query at the time of writing — expect one to follow, as it did for Gemma 4; flagging its
  absence rather than asserting it exists. [unsloth/MiniMax-M3-GGUF, Hugging Face](https://hf.co/unsloth/MiniMax-M3-GGUF)
