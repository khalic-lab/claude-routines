- **Switzerland votes tomorrow (Sun 14 June) on two federal measures — the SVP's "No 10-million Switzerland" cap and a tightening of the civil-service law.** The headline ballot is the Swiss People's Party (SVP) popular initiative officially titled the *Nachhaltigkeitsinitiative* ("sustainability initiative") but campaigned as **"Keine 10-Millionen-Schweiz"**: it would write into the constitution a ceiling barring the permanent resident population from passing 10 million before 2050, forcing immigration restrictions (and, as a backstop, termination of the EU free-movement agreement) if the trajectory exceeds the cap. The second measure is a revision of the **Civil Service Act (Zivildienstgesetz)** that aims to keep alternative civilian service (the legal substitute for compulsory military service) genuinely exceptional — chiefly by making it harder for men to switch to civil service *after* they have already completed a large part of their army obligation, which the military says is draining its ranks. The Federal Council and Parliament back the civil-service revision and oppose the SVP cap. A late-April SRG poll had the population-cap initiative at ~52% — within the margin where it could pass, which would be the first time a hard demographic ceiling enters the constitution and would directly collide with the Switzerland–EU free-movement regime. ## 🛡️ Cybersecurity

- **CVE-2026-11624 — Model Context Protocol DNS-rebinding exposure, CVSS 9.4 (critical).** The Model Context Protocol (MCP — the open standard, used by tools like this one, that lets AI assistants connect to external "servers" exposing tools and data) shipped without a way for a server to validate the **Origin/host** header on incoming connections. That gap enables **DNS rebinding** — an attack where a malicious web page tricks the victim's browser into treating the attacker's domain as `localhost`, so JavaScript on a random site can reach a locally-running MCP server the user never meant to expose. The fix landed in **v0.25.0**, which adds an `--allowed-hosts` flag alongside the existing `--allowed-origins`; both still default to `*`, but the server now prints a startup warning when left wide open. The CVSS 9.4 (critical; high confidentiality/integrity/availability impact, network vector, only "passive" user interaction needed) reflects that a developer running a local MCP server while browsing is the realistic victim. Practical takeaway: if you run MCP servers locally, upgrade and set explicit `--allowed-hosts`. ([NVD CVE-2026-11624](https://nvd.nist.gov/vuln/detail/CVE-2026-11624); ref: [google-mcp-toolbox issue #3113](https://github.com/googleapis/mcp-toolbox/issues/3113))

- **CVE-2026-35273 — Oracle PeopleSoft Enterprise PeopleTools missing-authentication flaw added to CISA's KEV catalog, with known ransomware use.** CISA added this to the **Known Exploited Vulnerabilities (KEV)** catalog — the federal list of bugs confirmed to be actively exploited in the wild — on 12 June, flagging it as **"Missing Authentication for Critical Function"** and, notably, **linked to known ransomware campaigns**. PeopleSoft PeopleTools underpins HR, finance and student-records systems at large enterprises and universities, so a missing-auth bug means an unauthenticated attacker can reach a function that should require login. The federal remediation deadline is **15 June** — unusually tight, signalling CISA sees real urgency. Patch per Oracle's alert immediately. ([NVD CVE-2026-35273](https://nvd.nist.gov/vuln/detail/CVE-2026-35273); [CISA KEV catalog](https://www.cisa.gov/known-exploited-vulnerabilities-catalog); [Oracle security alert](https://www.oracle.com/security-alerts/alert-cve-2026-35273.html))

- **ABRT/libreport local privilege-escalation chain — CVE-2026-54228 (CVSS 7.8), CVE-2026-54229 (7.0), CVE-2026-54230 (7.0), plus CVE-2026-54231.** Red Hat disclosed a cluster of local-privesc bugs in **ABRT** (Automatic Bug Reporting Tool — the crash handler that ships on Fedora/RHEL and collects core dumps) and its **libreport** library. They chain three classic primitives: a **TOCTOU race** (time-of-check-to-time-of-use — exploiting the gap between when a program checks a file and when it uses it) in the `abrt-dbus` `SetElement` method lets any local user write arbitrary files into the root-owned crash-dump directory (CVE-2026-54228); a second race in `ChownProblemDir` lets the attacker grab ownership of that directory while privileged scripts still hold a write lock (CVE-2026-54229); and a **symlink-following** bug in the post-create event scripts — they write output without `O_NOFOLLOW` — lets a swapped-in symlink redirect a root-owned write to **any file on the system**, i.e. arbitrary overwrite as root (CVE-2026-54230). Individually 7.x; chained, this is a credible local-user-to-root path on default Red Hat-family installs. All are local-access only (no remote vector), which caps the severity, but unprivileged local users are exactly the threat model on shared/multi-tenant Linux hosts. ([NVD CVE-2026-54228](https://nvd.nist.gov/vuln/detail/CVE-2026-54228); [CVE-2026-54229](https://nvd.nist.gov/vuln/detail/CVE-2026-54229); [CVE-2026-54230](https://nvd.nist.gov/vuln/detail/CVE-2026-54230); [Red Hat advisory](https://access.redhat.com/security/cve/CVE-2026-54228))

- **CVE-2026-11769 — Grafana Operator path-traversal / service-account-token theft, fixed in 5.24.0.** Grafana published an advisory (rated critical by the vendor; NVD CVSS 4.0 base 6.4) for its **Grafana Operator** (the Kubernetes controller that manages Grafana instances declaratively). The Operator evaluates **jsonnet** (a templating language) for dashboards inside the manager pod — so a user who can merely create `Dashboard` or `LibraryPanel` resources can craft jsonnet that exfiltrates the Operator's **Kubernetes service-account token**, a credential that typically carries broad cluster permissions. That's a tenant-to-cluster escalation in multi-team clusters. All versions **≤ 5.23** are affected; upgrade to **5.24.0**, or apply the published `ValidatingAdmissionPolicy` to block jsonnet-based dashboard resources as a stopgap. ([Grafana security advisory CVE-2026-11769](https://grafana.com/security/security-advisories/cve-2026-11769))
