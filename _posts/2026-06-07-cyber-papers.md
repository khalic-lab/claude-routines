## 🛡️ Cybersecurity

- **ComoDoS: a single crafted IPv6 packet remotely blue-screens any Windows box running Comodo Internet Security — CVE-2026-49494, CVSS 8.7 (High).** Researcher Marcus Hutchins (MalwareTech) publicly disclosed a zero-day after the vendor failed to respond. The bug is an _integer underflow_ (an arithmetic value dropping below zero and wrapping around to a huge number) in the IPv6 packet parser inside `Inspect.sys`, Comodo's firewall kernel driver. The parser subtracts each IPv6 extension-header's length from an unsigned 64-bit payload-length field without checking it first; feed it a packet whose declared payload length is smaller than its extension headers and the value wraps to ~18.4 quintillion (0xFFFFFFFFFFFFFFF8), causing an out-of-bounds read (and, on a second path, an oversized `memcpy`) in the Windows kernel — a crash (BSOD — the "blue screen of death" fatal kernel error). The kicker: because the firewall parses IPv6 _before_ it enforces any rules, the attack lands **even on a host with every port blocked** — there is no firewall configuration that stops it. CVSS 8.7 here means network-reachable, no authentication, no user interaction, high availability impact (RCE — remote code execution — is assessed unlikely). No patch exists. Hutchins chose the "Destination Options" extension header (next-header 60) precisely because routers rarely drop it, making remote exploitation reliable across the internet. Why it matters: a security product meant to protect you becomes the single remote kill-switch for the machine — and "all ports closed" gives no protection because the vulnerable code runs ahead of the firewall logic. ([NVD: CVE-2026-49494](https://nvd.nist.gov/vuln/detail/CVE-2026-49494); technical write-up [MalwareTech, 2026-06-03](https://malwaretech.com/2026/06/exploiting-a-remote-kernel-vulnerability-in-comodo-internet-security.html); PoC [GitHub/MalwareTech/ComoDoS](https://github.com/MalwareTech/ComoDoS))
