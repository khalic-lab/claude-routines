## 🛡️ Cybersecurity

Today's NVD window returned 70 published CVEs, 30 of them scored CVSS ≥ 7. The standouts:

- **CVE-2026-47065 — deserialization filter bypass in an Apache project, CVSS 9.8 (critical; remote, unauthenticated, leading to code execution).** `resolveProxyClass` is not overridden, so the `acceptMatchers` allow-list filter can be bypassed using a `java.lang.reflect.Proxy` in the serialized stream. In plain terms: Java deserialization (rebuilding an object from a byte stream sent over the wire) is a classic remote-code-execution vector, and the patch's safety filter — meant to only accept known-good classes — can be sidestepped by smuggling in a dynamic proxy class. The advisory marks the issue "fully addressed" in the fixed release. No known in-the-wild exploitation yet; patch available. ([NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-47065))
- **CVE-2026-35075 — hard-coded firmware password, CVSS 9.8 (critical; full device takeover, no auth).** An unauthenticated attacker can extract a default, hard-coded password straight from the firmware image and gain full access to every affected device. This is part of a cluster of flaws (see below) in what appears to be a building-automation / DALI lighting gateway, disclosed via Germany's VDE CERT (advisory VDE-2026-039). Hard-coded credentials are the worst kind of bug for fleet devices — you cannot rotate them, so the fix is a firmware update for every unit. ([NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-35075); [VDE-2026-039](https://www.certvde.com/en/advisories/VDE-2026-039/))
- **VDE-2026-039 cluster — stack overflows to root + file tampering in a DALI/building-automation gateway, CVSS up to 8.8.** Beyond the hard-coded password, the same device exposes a string of authenticated flaws: stack buffer overflows in the `gdv-serverconfig` and `dali-devconfig` services that give a low-privilege user full root (CVE-2026-35085, CVE-2026-35084, CVE-2026-35083), plus a family of `ugw-*` methods (the universal-gateway management API) that let an attacker read, delete, or restore arbitrary local files and kill processes (CVE-2026-35076 through CVE-2026-35082). DALI = Digital Addressable Lighting Interface, the protocol that controls commercial-building lighting; a compromised gateway is a quiet foothold inside an OT (operational-technology) network. No known exploitation; vendor fix via VDE advisory. ([NVD — CVE-2026-35085](https://nvd.nist.gov/vuln/detail/CVE-2026-35085); [VDE-2026-039](https://www.certvde.com/en/advisories/VDE-2026-039/))
- **CVE-2026-4035 — MLflow AI Gateway leaks secrets via environment-variable resolution, CVSS 9.1 (critical).** In MLflow before 3.11.0, the AI Gateway resolves environment variables embedded in stored "secrets," so an attacker who can register a secret can have the server expand `${ENV_VAR}` and exfiltrate values from the host's environment — exactly where API keys for downstream model providers tend to live. MLflow is the de-facto ML experiment-tracking and model-serving platform, so this sits squarely in the MLOps attack surface. Patched in 3.11.0; reported via a Huntr bounty (PoC-level, no confirmed in-the-wild use). ([NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-4035); [fix commit](https://github.com/mlflow/mlflow/commit/4a3f2f720cb4f058c9e0c5b883e0acc9ab64a7f3))
- **CVE-2026-5241 — arbitrary code execution via the LightGlue model-loading path in HuggingFace Transformers 5.2.0, CVSS 8.0 (high).** An attacker-controlled model repository can run arbitrary code when a victim loads a LightGlue (an image-feature-matching) model — the latest in a steady drip of "loading an untrusted model is code execution" bugs in the ML supply chain. Lesson for practitioners: treat a model checkpoint from an unknown source like an executable, not like data. Patched; reported via Huntr. ([NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-5241); [fix commit](https://github.com/huggingface/transformers/commit/676559d5022b74aaa0cee1cee0842b7f27c5320e))

Affects 10.0.4 through 10.0.24; fixed in 10.0.25. ([NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-42321); [GHSA-hwjc-8228-55x4](https://github.com/glpi-project/glpi/security/advisories/GHSA-hwjc-8228-55x4))

## 📄 ML research — second arXiv batch

Today's batch (arXiv RSS dated Wed, 03 Jun 2026; "new" announcements) leaned heavily toward efficient inference and Mixture-of-Experts internals. Six picks, deduped against this morning's Overview brief:

- **arXiv 2606.03458 (sub. 3 Jun 2026) — KVarN: variance-normalized KV-cache quantization.** `[preprint]` Why it matters: long "reasoning" generations are bottlenecked by the KV-cache (the stored keys/values the model reuses each step so it doesn't recompute the whole context). Quantizing that cache to fewer bits saves memory, but the paper shows that during autoregressive decoding the rounding errors _compound_ step-by-step — a failure mode prior work missed because it tested prefill-only. KVarN is calibration-free: a Hadamard rotation plus dual-axis variance normalization tames the outlier token-scales that drive the drift. A practical lever for running long-chain reasoning on less memory. ([abstract](https://arxiv.org/abs/2606.03458))
- **arXiv 2606.03928 (sub. 3 Jun 2026) — Value-aware stochastic KV-cache eviction for reasoning models.** `[preprint]` Why it matters: a complementary take on the same memory wall — instead of quantizing the cache, evict the least-useful entries. Two findings stand out: a tiny fraction of _value_ states have abnormally large magnitudes and evicting them is catastrophic (the model falls into repetitive reasoning loops), and adding randomness to which entries you drop actually improves accuracy by keeping the retained cache diverse. A concrete recipe for cheaper long reasoning without the usual accuracy cliff. ([abstract](https://arxiv.org/abs/2606.03928))
- **arXiv 2606.03002 (sub. 3 Jun 2026) — How quantization changes interpretable features: a sparse-autoencoder analysis.** `[preprint]` Why it matters: safety audits and "steering" interventions increasingly lean on SAE features (a sparse autoencoder decomposes a model's internal activations into human-interpretable concepts). We routinely quantize models for deployment and check only that perplexity/accuracy barely moves — but does the model still _compute the same way_? Using a frozen SAE as a fixed measuring stick, the authors test whether those interpretable features survive weight rounding. If they don't, an interpretability audit done on the full-precision model may not describe the quantized model you actually ship — a real gap for the safety-tooling pipeline. ([abstract](https://arxiv.org/abs/2606.03002))
- **arXiv 2606.03391 (sub. 3 Jun 2026) — When model merging breaks MoE routing: training-free calibration.** `[preprint]` Why it matters: model merging (fusing several fine-tuned models by arithmetic on their weights, no retraining) is a cheap way to combine skills, but it falls apart on Mixture-of-Experts models (MoE — only a few "expert" sub-networks fire per token, chosen by a learned router). The paper names the failure — "routing breakdown," where the merged router sends tokens to the wrong experts because the discrete top-k softmax is hypersensitive to weight perturbations — and offers a training-free fix to recalibrate it. Relevant as MoE becomes the default for frontier models. ([abstract](https://arxiv.org/abs/2606.03391))
- **arXiv 2606.03102 (sub. 3 Jun 2026) — Small RL controller, large language model: RL-guided adaptive sampling for test-time scaling.** `[preprint]` Why it matters: "test-time scaling" (sampling many candidate answers and picking the best) buys accuracy but burns compute and latency. The authors frame _when to stop sampling_ as a Markov decision process and train a lightweight reinforcement-learning controller to make that call per query, jointly trading off correctness, latency, and cost — replacing the hand-tuned heuristics most systems use today. A small, cheap policy steering an expensive model is an appealing pattern. ([abstract](https://arxiv.org/abs/2606.03102))
- **arXiv 2606.03825 (sub. 3 Jun 2026) — Dynamic short convolutions improve Transformers.** `[preprint]` Why it matters: a hybrid-architecture result. The paper adds input-dependent ("dynamic") short convolutions — small local filters whose weights depend on the input — to the query/key/value projections, preserving convolution's locality bias while gaining expressivity. It reports gains on hard associative-recall tasks (retrieving the right earlier token), one of the known weak spots that motivate state-space and conv-augmented alternatives to pure attention. ([abstract](https://arxiv.org/abs/2606.03825))
