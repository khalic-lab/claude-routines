## 🛡️ Cybersecurity

- **CVE-2026-45558 — Roxy-WI critical unauthenticated-adjacent RCE, CVSS 9.9.** Roxy-WI is a popular web UI for managing HAProxy, Nginx, Apache and Keepalived (load-balancer and reverse-proxy fleet management). In versions ≤ 8.2.6.4, the HAProxy "section-save" API endpoints accept a JSON `option` field that isn't validated before being written into the config, opening the door to code execution. The vector (`AV:N/AC:L/PR:L/UI:N/S:C`) means network-reachable, low-complexity, only *low* privileges required, and a **scope change** — i.e. a compromise of the web app cascades into the underlying host/proxies it manages. Companion flaws CVE-2026-45556, -45552 and -45550 (also 9.1–9.9) hit the same product. CVSS 9.9 here means: trivially exploitable by any authenticated low-priv user, full system compromise. No KEV listing yet (no confirmed in-the-wild exploitation), patch in the fixed release. [NVD CVE-2026-45558](https://nvd.nist.gov/vuln/detail/CVE-2026-45558)

- **Three fresh CISA KEV additions (added 2026-06-09) — patch-now list.** The KEV catalog (Known Exploited Vulnerabilities — CISA's authoritative list of bugs confirmed exploited in the wild, with federal patch deadlines) gained: **CVE-2026-11645**, a Google Chromium V8 out-of-bounds read/write allowing arbitrary code execution *inside the renderer sandbox* via a crafted web page (affects all Chromium-based browsers — Chrome, Edge, Brave); **CVE-2026-7473**, an Arista EOS flaw where the switch incorrectly decapsulates and forwards unexpected tunnelled packets (a routing/segmentation-bypass class — an attacker can smuggle traffic past network boundaries); and **CVE-2026-20245**, a Cisco Catalyst SD-WAN Manager output-escaping flaw letting an authenticated local attacker run arbitrary commands. Why it matters: KEV entries mean active exploitation is confirmed, so these jump the queue ahead of higher-CVSS-but-theoretical bugs. - **Ghidra (NSA's open-source reverse-engineering suite) gets a cluster of server-mode flaws, CVSS 8.7 each.** Three issues land in Ghidra's *shared-server* features (used by reverse-engineering teams collaborating on a shared project repo backed by PostgreSQL): **CVE-2026-52758** (SQL injection — user input concatenated straight into SQL via the BSim network query protocol, letting a remote attacker read/modify/delete the backing database); **CVE-2026-52754** (authentication bypass — the PKI module accepts a valid CA-signed certificate with a *null signature*, so anyone holding another user's public cert can impersonate them and escalate); and **CVE-2026-49498** (SQLi in the password-change path via unescaped double-quotes in usernames). Why it matters: malware analysts are exactly the population that runs untrusted-adjacent tooling on sensitive networks; a shared Ghidra server is a juicy lateral-movement target. ## 📄 ML research — second arXiv batch

_All papers below are from the 2026-06-09 US-time arXiv submission window (announced into the 2026-06-10 cycle), distinct from this morning's Overview batch (which cited 2606.09850/863/864/874, 2606.10324/10528)._

- **`arXiv:2606.11052` (2026-06-09) — Chain-of-thought fine-tuning quietly destroys long-range memory in hybrid models.** Hybrid LLMs mix cheap linear-attention layers (which compress the whole past into a fixed-size running state, à la Mamba/SSMs) with a few full-attention layers, to dodge the quadratic cost of standard attention. The authors find that CoT (chain-of-thought — training a model to write out intermediate reasoning steps) supervised fine-tuning systematically *degrades* these models' ability to recall facts from far back in the context — "attention amnesia" — and propose a fix. Why it matters: hybrid architectures are the leading bet for cheap long-context inference, so a finding that the most popular reasoning-training recipe silently breaks their headline feature is a real design constraint. `[preprint]` [abs](https://arxiv.org/abs/2606.11052)

- **`arXiv:2606.11164` (2026-06-09) — Give reasoning models a *KV-cache budget* and spend it where it counts.** Long chain-of-thought traces blow up the KV cache (the stored keys/values a transformer keeps so it needn't recompute past tokens — it grows linearly with output length and dominates memory for long reasoning). ReasonAlloc allocates a fixed cache budget *hierarchically* across the decode, keeping more of the tokens that matter and evicting the rest, rather than applying one eviction rule uniformly. Why it matters: reasoning models are expensive precisely because they generate enormous intermediate output; smarter cache budgeting is a direct lever on serving cost. `[preprint]` [abs](https://arxiv.org/abs/2606.11164)

- **`arXiv:2606.11172` (2026-06-09) — Steer a reasoning model better by first *predicting* what it's about to do.** "Test-time steering" nudges a model's behaviour by editing its hidden activations mid-generation, but doing so blindly often hurts output quality. This paper argues good steering implicitly needs a forecast of the model's *future* behaviour, and shows that explicitly predicting it yields cleaner control with less quality loss. Why it matters: steering is a leading interpretability-adjacent tool for making deployed models safer/more controllable on the fly; tying it to a predictive signal makes it less of a blunt instrument. `[preprint]` [abs](https://arxiv.org/abs/2606.11172)

- **`arXiv:2606.11119` (2026-06-09) — TRACE: stop wasting compute on agentic RL rollouts.** RLVR (reinforcement learning with verifiable rewards — training on tasks where correctness can be checked automatically, e.g. unit tests pass) is bottlenecked by *rollouts*: the agent must actually attempt many trajectories to learn, and most attempts on easy or hopeless tasks waste compute. TRACE is a unified framework for allocating the rollout budget across tasks so the expensive trial-and-error concentrates where it teaches the most. Why it matters: rollout cost is the dominant expense in training agentic models, so principled budget allocation is a straight efficiency win. `[preprint]` [abs](https://arxiv.org/abs/2606.11119)

- **`arXiv:2606.11046` (2026-06-09) — Turning an aligned model into a "reasoning" model can erode its safety.** Labs routinely convert instruction-tuned LLMs into reasoning models via extra post-training, optimising for multi-step accuracy. This paper audits whether that conversion *preserves alignment* (the model's trained refusals and safety behaviour) and finds it often doesn't — accuracy-focused reasoning training degrades trustworthiness. Why it matters: it's a direct complement to this morning's finding that KV-cache quantization can erase refusals — both say the same uncomfortable thing, that routine capability-boosting steps can silently re-open safety holes the benchmark never measures. `[preprint]` [abs](https://arxiv.org/abs/2606.11046)

- **`arXiv:2606.11150` (2026-06-09) — ABC-Bench: measuring how good AI agents are getting at wet-lab-relevant biology.** As LLM agents move from summarising papers to *running in-silico biology* (computational experiments — sequence design, structure prediction, protocol planning), the dual-use risk rises. ABC-Bench is an "agentic bio-capabilities" benchmark aimed at biosecurity: it quantifies where current agents stand on the capabilities that matter for that risk. Why it matters: you can't govern a capability you can't measure, and biosecurity evaluation has lagged far behind the general-agent benchmarks — this is an attempt to close that gap with numbers rather than vibes. `[preprint]` [abs](https://arxiv.org/abs/2606.11150)
