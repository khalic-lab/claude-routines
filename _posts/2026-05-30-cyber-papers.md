The day's signal is on the CISA KEV (Known Exploited Vulnerabilities — CISA's list of flaws confirmed under active attack) side and in two software-supply-chain incidents.

- **Palo Alto PAN-OS GlobalProtect authentication bypass now actively exploited — CVE-2026-0257.** CISA added this to KEV on 29 May, confirming real-world exploitation. The flaw lets an unauthenticated attacker bypass authentication on the GlobalProtect portal/gateway (PAN-OS's remote-access VPN front-end) and establish an unauthorized VPN connection — i.e. a foothold into the corporate network behind the firewall. NVD rates it CVSS 9.1 (critical) under the v3.1 scoring system; Palo Alto's own CVSS v4.0 vector is 7.8 (high). Panorama and Cloud NGFW are unaffected; patch and check for unauthorized GlobalProtect sessions. [NVD CVE-2026-0257](https://nvd.nist.gov/vuln/detail/CVE-2026-0257) · [Palo Alto advisory](https://security.paloaltonetworks.com/CVE-2026-0257) [ongoing since 2026-05-30 — flagged earlier today in the weekend brief; included here as the priority patch, no new development since]

- **Nx Console supply-chain compromise — CVE-2026-48027 (KEV-listed 27 May).** A malicious build of Nx Console (the IDE companion extension for the Nx/Lerna JavaScript monorepo build tools) — version 18.95.0 — was published to the Visual Studio Marketplace at 12:30 UTC on 19 May and pulled ~18 minutes later at 12:48 UTC; the OpenVSX copy lingered longer before detection. This is a classic "publish-poisoned-package-then-yank" supply-chain attack: even an 18-minute window is enough for auto-updaters to pull the trojanised extension onto developer machines, which then run with the developer's local privileges. Nx's postmortem lists indicators of compromise; anyone who auto-updated VS Code/Cursor extensions on 19 May should check. [NVD CVE-2026-48027](https://nvd.nist.gov/vuln/detail/CVE-2026-48027) · [Nx postmortem & IoCs](https://nx.dev/blog/nx-console-v18-95-0-postmortem#indicators-of-compromise) · [GHSA-c9j4-9m59-847w](https://github.com/nrwl/nx-console/security/advisories/GHSA-c9j4-9m59-847w)

- **Second supply-chain entry: Daemon Tools Lite shipped embedded malicious code — CVE-2026-8398 (KEV-listed 27 May).** CISA added a Daemon Tools Lite (a popular Windows disk-image/virtual-drive utility) build carrying embedded malicious code, the same week as the Nx incident — two distinct "trusted installer was poisoned" cases hitting KEV together. The pattern is the takeaway: attackers increasingly compromise the distribution channel rather than exploit a code bug, because users grant installers full trust. [CISA KEV catalog](https://www.cisa.gov/known-exploited-vulnerabilities-catalog) [single-source]

- **Spectra (Gutenberg Blocks) WordPress plugin — authenticated RCE, CVE-2026-7465, CVSS 8.8.** All versions up to and including 2.19.25 of this widely installed block-editor/page-builder plugin allow remote code execution (RCE — running attacker-chosen code on the server) by an authenticated user with merely Contributor-level access. The exploit is elegant and worth understanding: the attacker embeds a two-block payload in post content — the first block registers a fake `uagb/`-prefixed block type whose `render_callback` points at attacker-chosen code, and the second block of that fake type triggers it via PHP's `call_user_func()` during the page's sequential block rendering. Because Contributor is a low-trust role on many multi-author sites, this meaningfully lowers the bar from "admin compromise" to "any contributor." Update past 2.19.25. [NVD CVE-2026-7465](https://nvd.nist.gov/vuln/detail/CVE-2026-7465)

- **Recent KEV additions worth a patch-cycle scan.** Beyond the above, CISA's catalogue added this past week: **Drupal core SQL injection** CVE-2026-9082 (22 May) — SQLi (injecting attacker SQL into a query) in the popular CMS core; **Trend Micro Apex One (on-prem) directory traversal** CVE-2026-34926 (21 May) — path traversal in an endpoint-protection product, an ironic foothold given it's a security tool; and **LiteSpeed cPanel plugin privilege escalation** CVE-2026-48172 (26 May), relevant to shared-hosting fleets.
