- **CVE-2026-42897 — Microsoft Exchange Server OWA XSS zero-day (CVSS 8.1 / NVD 6.1, actively exploited, no patch).** An improper-neutralisation (CWE-79) flaw in Outlook Web Access allows a remote attacker to execute arbitrary JavaScript in a victim's browser by sending a specially crafted email that the target merely needs to open in OWA. Successful exploitation can yield session token theft, mailbox access, and unauthorised changes to mail settings. Affects Exchange Server 2016, 2019, and Subscription Edition on-premises only; Exchange Online is unaffected. Microsoft confirmed active exploitation in a May 14 advisory and published two interim mitigations (URL-rewrite rule + enhanced input filtering); no patch yet. CISA added to the Known Exploited Vulnerabilities catalog on 15 May 2026 with a federal remediation deadline of 29 May. Exploitation status: **active, weaponised via email lure**. - **Grafana Labs GitHub breach via TanStack npm supply-chain attack; source code exfiltrated.** Grafana Labs confirmed on 19 May that its entire GitHub codebase — including private repositories with internal operational details — was downloaded by attackers following a supply-chain compromise through the TanStack npm package (orchestrated by the threat actor group TeamPCP, the same campaign that also breached OpenAI and Mistral AI). - **CVE-2026-32202 — Windows Shell zero-click NTLM coercion (APT28, actively exploited).** A protection-mechanism failure in Windows Shell stemming from an incomplete fix for CVE-2026-21510: when a user opens any folder containing a malicious `.LNK` file — no double-click required — Windows automatically initiates an SMB connection to an attacker-controlled server, sending the victim's Net-NTLMv2 hash. In environments without SMB signing, that hash is immediately relayable for domain-wide access. APT28 (Fancy Bear / Forest Blizzard) has exploited the flaw in campaigns targeting Ukraine and EU nations since at least December 2025, distributing malicious shortcuts via shared drives and phishing. Microsoft patched on April 14 without marking the advisory "Exploited"; CISA and Microsoft corrected the omission on April 27 and added to KEV. Recommended mitigations: apply the April 14 patch; block outbound SMB at network perimeters; enforce SMB signing. Exploitation status: **active, attributed APT28**. - **CVE-2026-20182 — Cisco Catalyst SD-WAN auth bypass (CVSS 10.0, KEV, UAT-8616).** An authentication bypass in the SD-WAN peering mechanism (vdaemon, DTLS UDP port 12346) allows an unauthenticated remote attacker to claim vHub device identity, bypassing certificate verification and obtaining full admin access to the SD-WAN Controller (formerly vSmart) and Manager (formerly vManage). From that position, attackers can read and modify SD-WAN fabric configurations, establish persistent access via NETCONF, and pivot across the entire network fabric. Active exploitation has been attributed by Cisco Talos to UAT-8616, a sophisticated actor targeting Cisco SD-WAN infrastructure since at least 2023. CISA's federal-agency remediation deadline was 17 May 2026; civilian agencies that have not patched should treat this as overdue. - **CVE-2026-31431 — Linux kernel "Copy Fail" local privilege escalation (CVSS 7.8, KEV since 1 May).** A high-severity LPE flaw affecting all major Linux distributions allows an unprivileged local user to obtain root via a race condition in the kernel's copy-on-write path; impact is amplified in cloud and Kubernetes environments where multi-tenant workloads share a kernel. CISA added to KEV 1 May 2026 with a mandatory federal remediation deadline of 15 May; Microsoft published detailed technical analysis under the name "Copy Fail." The federal deadline has passed — organisations that have not patched containerised and cloud Linux workloads are now overdue.
