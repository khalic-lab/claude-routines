## 🛡️ Cybersecurity

- **Laravel-Lang PHP supply chain attack: 700+ versions backdoored with RCE credential stealer.** On 22 May 2026, a threat actor with push access to the Laravel-Lang GitHub organisation rewrote every git tag across four popular Composer packages within a 15-minute window, pointing legitimate-looking release tags to commits inside a malicious fork: `laravel-lang/lang` (7.8k stars), `/attributes`, `/http-statuses`, and `/actions`. 233 package versions were compromised. The payload — a `src/helpers.php` registered in `composer.json` under `autoload.files` — executes automatically on every PHP request and systematically harvests AWS/GCP/Azure/DigitalOcean access keys, Kubernetes profiles, Docker tokens, HashiCorp Vault secrets, SSH private keys, Git credentials, browser passwords, cryptocurrency wallets, and password-manager databases. **Immediate action:** any `composer install` of these packages on or after 2026-05-22 22:32 UTC should be treated as a full credential compromise — rotate CI tokens, cloud keys, GITHUB_TOKEN, deploy keys, and database credentials immediately. [*The Hacker News*, May 2026](https://thehackernews.com/2026/05/laravel-lang-php-packages-compromised.html) / [*StepSecurity*, May 2026](https://www.stepsecurity.io/blog/laravel-lang-supply-chain-attack)

Active exploitation confirmed. Fixed in version 2.4.5. Operators should upgrade immediately and audit for unauthorised script execution in access logs. - **NGINX Rift — CVE-2026-42945 (CVSS v4: 9.2): 18-year-old heap overflow under active exploitation since May 16.** Disclosed 13 May by F5 and depthfirst, this heap buffer overflow in `ngx_http_rewrite_module` (present in all NGINX builds 0.6.27–1.30.0 and NGINX Plus R32–R36) stems from a two-pass contract violation: an `is_args` flag set during the length-calculation phase leaks into the copy phase, causing `ngx_escape_uri` to write past the allocated buffer when a rewrite rule combines an unnamed PCRE capture with a `?` character.
