## 🚀 New models, datasets, & open weights

The dominant story of the past several weeks in open weights is a Chinese model cluster: four labs released frontier-class agentic coding models in a 12-day window in late April–early May, all reaching roughly the same capability ceiling at 25–250× lower inference cost than Western closed frontier models.

- **DeepSeek V4 Pro / Flash** (DeepSeek, rel. ~Apr 24) — MIT license; V4 Pro is a dense Transformer with 1M-token context window. V4 Pro permanently repriced on May 22 to **$0.435 input / $0.87 output per million tokens** — making hosted access roughly 30× cheaper than Claude Opus 4.7 at comparable agentic coding capability. On Artificial Analysis Intelligence Index v4.0, V4 Pro scores 52. Simon Willison documented the V4 release on April 24: ["DeepSeek V4—almost on the frontier, a fraction of the price" [via snippet]](https://simonwillison.net/2026/apr/24/deepseek-v4/); Artificial Analysis benchmark coverage: [artificialanalysis.ai [via snippet]](https://artificialanalysis.ai/articles/deepseek-is-back-among-the-leading-open-weights-models-with-v4-pro-and-v4-flash)

- **Kimi K2.6** (Moonshot AI, rel. Apr 20) — Open weights; 1 trillion total parameters, 32B activated per token, MoE architecture (384 experts, 8+1 selected), 256K context, MIT license. Adds native multimodal support (MoonViT 400M vision encoder accepting image and video) and extends Agent Swarm system to 300 sub-agents executing up to 4,000 coordinated steps. Benchmark highlights: SWE-bench Verified 80.2%, SWE-bench Pro 58.6%, LiveCodeBench v6 89.6%, Terminal-Bench 2.0 66.7%. Artificial Analysis Intelligence Index ranks it at 54, highest among the open-weight cluster. Model card: [NVIDIA Build / Moonshot AI [via snippet]](https://build.nvidia.com/moonshotai/kimi-k2.6/modelcard)

- **GLM-5.1** (Z.ai / Zhipu, rel. Apr 7) — MIT license; 754B total parameters, MoE architecture (~40B active), 131K context. Positions itself as the sovereignty pick for organizations needing to run locally: fits on 8× H100 via vLLM, MIT license allows full commercial use without restriction. SWE-bench Pro 58.4% (competitive with Kimi K2.6), state-of-the-art on NL2Repo and Terminal-Bench 2.0. Model card: [HuggingFace zai-org/GLM-5.1](https://huggingface.co/zai-org/GLM-5.1)

- **MiniMax M2.7** (MiniMax, rel. ~May 2026) — MoE architecture, 229.9B total / 9.8B activated parameters, 204.8K context, open weights. Optimized for agentic coding and long-context tasks; Artificial Analysis composite score ranks it #1 among open-source models globally on its index. The technical report for the MiniMax-M2 series (covering architecture and training methodology) was published today on arXiv (2605.26494). API pricing $0.255 input / $1.00 output per million tokens. Model page: [MiniMax AI](https://www.minimax.io/news)

- **Grok Build 0.1** (xAI, rel. May 20) — see Lab blogs section above for full detail. Open-weights availability not confirmed at publication time; API access at $1.00/$2.00 per million tokens. [OpenRouter listing [via snippet]](https://openrouter.ai/x-ai/grok-build-0.1)

**Four-model context**: On neutral testing (Artificial Analysis Intelligence Index v4.0), Kimi K2.6 leads open weights at 54, DeepSeek V4 Pro at 52, GLM-5.1 at 51, MiniMax M2.7 close behind. At May 22 pricing, the entire cluster sits at $0.25–$0.90 per million input tokens for hosted API access versus $5.00 for Claude Opus 4.7 — a 6–20× gap. The pricing convergence and capability parity are the most consequential structural shift in the LLM market this year.

## 🍎 Apple Silicon / on-device

- **llama.cpp b9370 (May 27): Hexagon backend gains Q4_1 MUL_MAT and Q8_1 vector dot product** — The May 27 release adds `Q4_1` support in `MUL_MAT` and `MUL_MAT_ID` operations for the Hexagon (Qualcomm AI) backend, switches vector dot products to dynamic Q8_1 quantization, fixes a buffer overflow in repack operations, and adds early-wake + operation-batch polling for latency improvement. These changes allow the Hexagon backend to claim a substantially larger share of computational graphs, reducing CPU idle time. Note: this targets Qualcomm Snapdragon on-device hardware, not Apple Silicon directly. A same-day minor release also fixed an SSL log message in `llama-server` that incorrectly displayed `http` instead of `https` when the server started with SSL key and cert. Both are maintenance-class releases. — [github.com/ggml-org/llama.cpp/releases](https://github.com/ggml-org/llama.cpp/releases/tag/b9370) _(direct fetch)_
