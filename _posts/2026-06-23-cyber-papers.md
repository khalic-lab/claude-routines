Today's NVD batch (74 CVEs) was, however, unusually heavy on AI/agent tooling.

- **FOSSBilling: an unauthenticated auth-bypass chains with a template-injection bug into full RCE — CVE-2026-27604, CVSS 10.0 (maximum; trivially exploitable, complete system compromise).** In this open-source billing/client-management platform (versions 0.5.4 through 0.7.x), an authorization flaw lets anyone reach the privileged `/api/system/*` endpoints, which resolve to the cron *admin* identity — so an attacker invokes admin API methods with no credentials, session, or CSRF token. Paired with a separate Server-Side Template Injection (SSTI — injecting code into an unsandboxed Twig template engine; CVE-2026-28496, CVSS 9.4), that yields remote code execution. Fixed in 0.8.0; interim mitigations are blocking `/api/system/*` at the proxy/WAF and IP-allowlisting the API. [NVD CVE-2026-27604](https://nvd.nist.gov/vuln/detail/CVE-2026-27604) · [NVD CVE-2026-28496](https://nvd.nist.gov/vuln/detail/CVE-2026-28496)
- **A cluster of same-day CVEs hit popular AI-agent infrastructure — the orchestration layer many developers are wiring up right now.** *Flowise* (low-code LLM-app builder) before 3.1.2 has OS command injection in its Custom MCP Server feature: an incomplete blocklist (e.g. `docker build` and `npx --yes` slip past) lets any logged-in user run arbitrary host commands (CVE-2026-56274, CVSS 8.7). *n8n* (workflow automation) before 2.25.7/2.26.2 exposes its `@n8n/mcp-browser` MCP endpoint in HTTP mode with **no authentication**, so any reachable site/client can drive browser controls — navigation, JavaScript evaluation, cookie/storage access (CVE-2026-54309, CVSS 8.8). *Crawl4AI* before 0.8.8 allows unauthenticated arbitrary file write via symlink/TOCTOU (time-of-check-to-time-of-use — a race between validating and using a path) on its screenshot/PDF endpoints (CVE-2026-56258, CVSS 9.2). MCP (Model Context Protocol — the emerging standard for connecting LLMs to tools) is becoming a real attack surface. [NVD CVE-2026-56274](https://nvd.nist.gov/vuln/detail/CVE-2026-56274) · [NVD CVE-2026-54309](https://nvd.nist.gov/vuln/detail/CVE-2026-54309) · [NVD CVE-2026-56258](https://nvd.nist.gov/vuln/detail/CVE-2026-56258)
- **picklescan can again be tricked into passing malicious model files — CVE-2026-56315, CVSS 9.3 (critical).** picklescan is the scanner that's supposed to catch booby-trapped Python `pickle` files (the format many ML model checkpoints ship in, which can execute code on load). Versions before 1.0.4 fail to block at least seven standard-library modules (e.g. `uuid`, `imaplib`, `_aix_support`) that expose functions giving direct command execution, so a crafted checkpoint achieves RCE while the scanner reports it clean. This is a *distinct* CVE continuing the picklescan-bypass saga that produced CVE-2026-53874 last week — same scanner, new evasion class, not the same flaw. Anyone loading third-party model weights should treat picklescan as defence-in-depth, not a guarantee. [NVD CVE-2026-56315](https://nvd.nist.gov/vuln/detail/CVE-2026-56315)
- **ManageEngine generates predictable SSO tickets, enabling account takeover — CVE-2026-11374, CVSS 9.0.** In ADSelfService Plus, RecoveryManager Plus, M365 Manager Plus and ADAudit Plus, the single-sign-on tickets used to authenticate a session can be *predicted* by an unauthenticated attacker (insufficient randomness), allowing them to hijack accounts — a serious flaw given these products sit at the identity/password-reset layer of Windows/AD environments. Vendor advisory and patches available. [NVD CVE-2026-11374](https://nvd.nist.gov/vuln/detail/CVE-2026-11374)
- **Zephyr RTOS Bluetooth stack: malformed packets crash or corrupt memory — CVE-2026-10658 and CVE-2026-10651 (CVSS 7.1 each).** Zephyr is a widely embedded real-time OS for IoT/microcontrollers. A missing length check in the Bluetooth ISO receive path (`bt_iso_recv()`) lets malformed HCI ISO data trip an assertion/over-read, and a malformed Bluetooth Classic SDP attribute reaches an assertion in the SDP parser — both remotely triggerable by an in-range attacker, a denial-of-service (and memory-safety) risk for connected devices that are rarely patched. ## 📄 ML research — second arXiv batch

_The category RSS feeds were empty this announcement cycle (as they were this morning), so candidates were drawn from the date-filtered Atom API; all items below were submitted 2026-06-22 (the batch announced for 2026-06-23, i.e. "yesterday US-time" per the rollover rule). The six papers cited in today's Morning Overview are excluded._

- **arXiv:2606.23581** (submitted 2026-06-22) — *Kamera: Unified Position-Invariant Multimodal KV Cache for Training-Free Reuse.* Why interesting: multimodal agents keep re-examining the same frames/screenshots, but the KV-cache (key-value cache — the stored attention state that lets a model avoid recomputing earlier tokens) only serves reuse at a fixed leading position, so look-backs re-encode from scratch. The paper pins down what naive reuse loses — the cross-chunk conditioning a chunk absorbed from neighbours, which is invisible to single-hop recall but is exactly what multi-hop reasoning needs (blind reuse halves multi-hop accuracy) — and repairs it with a small training-free low-rank "patch" stored per chunk, recovering full accuracy at a fraction of the cache footprint. [abstract](https://arxiv.org/abs/2606.23581) [preprint]
- **arXiv:2606.23595** (submitted 2026-06-22) — *SPIRAL: Learning to Search and Aggregate.* Why interesting: models are trained to reason in a single sequential chain, but at test time we often scale compute three ways — longer chains, many parallel samples, and aggregating those samples. SPIRAL trains all three end-to-end with reinforcement learning (RL — learning from a reward signal rather than labelled answers), so the model learns to emit a *set* of traces collectively useful to an aggregator it also learns; reported up to 11× better scaling efficiency and 15% higher performance than GRPO (a standard RL-for-reasoning baseline). [abstract](https://arxiv.org/abs/2606.23595) [preprint]
- **arXiv:2606.23590** (submitted 2026-06-22) — *The Topology of Ill-Posed Questions: Persistent Homology for Detection and Steering in LLMs.* Why interesting: ambiguous, underspecified or contradictory questions have no single right answer, yet models answer anyway. The authors treat a prompt's hidden states at each layer as a point cloud and summarise its *shape* with persistent homology (a tool from topological data analysis that measures how clusters merge across scale), getting three compact per-layer numbers. Those topology features beat prompt- and hidden-state baselines at flagging ill-posedness (e.g. 67.4%→78.9% on AmbigQA) and can *steer* the model toward asking for clarification or abstaining — interpretability that turns into a control knob. [abstract](https://arxiv.org/abs/2606.23590) [preprint]
- **arXiv:2606.23524** (submitted 2026-06-22) — *Scaling State-Space Models from Lines to Paragraphs: An Ablation of Mamba-based OCR.* Why interesting: Transformer attention costs grow quadratically with text length, a problem for transcribing whole paragraphs. State-space models (SSMs like Mamba — a sequence architecture with linear-time decoding) are the leading efficient alternative. On clean synthetic paragraphs the Mamba OCR matches a Transformer's accuracy while running 1.4–4.5× faster (the gap widening with length), but on *real handwriting* it lags clearly (8.2% vs 4.2% character-error-rate on IAM lines) — a gap the authors trace to data-hunger, not an intrinsic architectural ceiling. A useful, honest "when SSMs do and don't pay off" result. [abstract](https://arxiv.org/abs/2606.23524) [preprint]
- **arXiv:2606.23607** (submitted 2026-06-22) — *Scaling Linear Mode Connectivity and Merging to Billion Parameter Pretrained Transformers.* Why interesting: "linear mode connectivity" asks whether two independently trained networks can be connected by a straight line in weight space along which loss stays low — if so, you can *merge* models by averaging. Prior methods don't scale; this one applies functionality-preserving weight transformations and a dual-learning procedure so both models bend toward a shared path, reporting near-zero loss barriers at medium scale (a first) and only small barriers for billion-parameter LLMs. Practically, cleaner merging means combining fine-tuned models without retraining. [abstract](https://arxiv.org/abs/2606.23607) [preprint]
- **arXiv:2606.23671** (submitted 2026-06-22) — *Can LLMs Reliably Self-Report Adversarial Prefills, and How?* Why interesting: a safety hope is that a model can notice when it's been jailbroken. Across ten open instruction-tuned models (3B–70B), none reliably recognise that their own prior output was coerced by an adversarial *prefill* (an attack that puts compliant text in the model's mouth to bootstrap a harmful continuation) — they claim genuine intent on prefilled answers 27.3% of the time on average. The introspective signal traces largely to the model's refusal machinery, and fine-tuning to sharpen self-report counterintuitively *raised* attack success — a sobering caution against trusting LLM self-reports in safety pipelines. [abstract](https://arxiv.org/abs/2606.23671) [preprint]
